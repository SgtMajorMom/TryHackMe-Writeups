## 📌 Overview
This module introduces the fundamentals of securing AI and machine learning systems. It covers how AI components expand the attack surface, how adversaries target AI/ML pipelines, and how to apply structured threat modelling to identify risks and recommend mitigations.

This write-up summarizes the key concepts, security considerations, and defender-focused techniques explored throughout the module.

---

## 🎯 Learning Objectives
- Understand AI systems as an attack surface  
- Identify weaknesses in AI/ML pipelines  
- Recognize LLM-specific security threats  
- Apply STRIDE, OWASP, and ATLAS frameworks  
- Perform reconnaissance on AI systems  
- Conduct an AI threat modelling assessment  

---

## 🛡️ Module Breakdown

### **1. Securing AI Systems**
- AI architecture and trust boundaries  
- Secure design principles  
- Identifying attack surfaces using OWASP/ATLAS  

### **2. LLM Security**
- Prompt manipulation  
- Hallucinations  
- Data leakage  
- Misuse of model capabilities  

### **3. AI Threat Modelling**
- STRIDE  
- OWASP  
- ATLAS  
- Evaluating inputs, outputs, pipelines, and integrations  

### **4. AI System Reconnaissance**
- Discovering ML services  
- Identifying frameworks  
- Metadata exposure  
- Reviewing endpoints  

### **5. Threat Modelling Assessment**
- Identifying threats  
- Mapping trust boundaries  
- Recommending mitigations  

---

## 🧩 Key Takeaways
- AI introduces new attack surfaces  
- LLMs require specialized guardrails  
- Threat modelling is essential for AI security  
- Metadata exposure is a common risk  
- Secure AI requires architecture, controls, monitoring, and governance
